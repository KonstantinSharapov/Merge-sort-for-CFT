1)Программа сырая... спустя стоко времени...
2)Можетделать сортировку чисел 10>2 и строк "2">"10". 
3)AppMerge.exe запускается от внешней библиотеки которая у вас на компьюторе, или от вложеной jre 
  которая находиться вместе с AppMerge.exe в одной папке
4)технические особенности AppMerge.exe
  а) sdk 8, jre 8 
  б) собран проект от artifacts через intelij idea, не через maven
5)Корректный запуск программы
  а)для чисел: -i
  б)для строк: -s
  в)запись файлов по возрастанию: -a
  г)запись файлов по убыванию:    -d
  д)выходной файл: out.txt (Если файла нет, то его создаст программа, иначе она просто перезапишет существующий файл)
  е)входные файлы: int1.txt int2.txt ... intN.txt или str1.txt str2.txt ... strN.txt (имя входного файла может быть любым)
  ж)диапазон чисел которая может принимать на вход программа: 0 <= N <= 10^9    

  з)пример запуска:
    сортировка чисел по возрастанием: AppMerge.exe -i -a out.txt 1.txt 4.txt 2.txt
    сортировка строк по убыванию:     AppMerge.exe -s -d out.txt 1.txt 4.txt 2.txt
  
  и)При вводе входных файлов нужно указывать верные имена тех фалов которые будет читать программа и сортировать.
    То есть есть: AppMerge.exe -i -a out.txt int1.txt int4.txt int2.txt 5.txt
    Если 5.txt не существует или имя данного файла который находитсья в папке не верно, то программа выдаст ошибку.   

P.S при работе с программой важно что бы на компе была установленна внешняя бибилиотека, если этого нет, то:
    нужно скачать из папки work space папку jre и положить этут папку в одно место вместе с программой.
    Проще будет просто скачать папку work space, там будет внутренняя библиотека, сама программа и рабочий материал.
