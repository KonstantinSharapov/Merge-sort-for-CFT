3)AppMerge.exe запускается от внешней библиотеки которая у вас на компьюторе, или от вложеной jre 
  которая находиться вместе с AppMerge.exe в одной папке
4)технические особенности AppMerge.exe
  а) sdk 8, jre 8 
  б) собран проект от artifacts через intelij idea, не через maven
5)Корректный запуск программы
  а)для чисел: -i
  б)для строк: -s
  в)запись файлов по возрастанию: -a
  г)запись файлов по убыванию:    -d
  д)выходной файл: out.txt (Если файла нет, то его создаст программа, иначе она просто перезапишет существующий файл)
  е)входные файлы: int1.txt int2.txt ... intN.txt или str1.txt str2.txt ... strN.txt (имя входного файла может быть любым)
  ё)при запуске для чисел важно учитывать, что бы в входных данных не было просто строк со символами, инчае будет ошибка, 
    то есть пример:
    
    1%;*0 - это прочтеться как 10
    %;*   - это выдаст ошибку
    a     - это выдаст ошибку
    1a    - это прочтеться как 1
    диапазон чисел которая может принимать на вход программа: 0 <= N <= 10^9    

  ж)пример запуска:
    сортировка чисел с возрастанием: AppMerge.exe -i -a out.txt int1.txt int4.txt int2.txt
    сортировка строк с по убыванию:  AppMerge.exe -s -d out.txt str1.txt str4.txt str2.txt
  
  з)При вводе входных файлов нужно указывать верные имена тех фалов которые будет читать программа и сортировать.
    То есть есть: AppMerge.exe -i -a out.txt int1.txt int4.txt int2.txt 5.txt
    Если 5.txt не существует или имя данного файла который находитсья в папке не верно, то программа выдаст ошибку.
